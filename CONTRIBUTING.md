# Contributing Guide

Thank you for your interest in contributing to Awakened Imagination! This document outlines the guidelines and processes for contributing to the project.

## Project Overview

Awakened Imagination is a visionary space development initiative focused on orbital stewardship, interplanetary foundations, planetary life support, and global prosperity. The project is implemented as a static website with HTML, CSS, and JavaScript.

## How to Contribute

### 1. Reporting Issues

If you encounter a bug or have a feature request:

- **Search existing issues** to avoid duplicates
- **Provide clear steps to reproduce** for bugs
- **Include relevant details** like browser version, operating system, and expected vs. actual behavior
- **Use descriptive titles** summarizing the issue

### 2. Making Changes

#### Setting Up Locally

1. Clone the repository:
   ```bash
   git clone https://github.com/genidma/asi.surge.sh.git
   cd asi.surge.sh
   ```

2. No build step is required for this static site - just open the HTML files in a browser.

#### Code Style

- Follow the existing CSS conventions in `mystyles.css`
- Use semantic HTML5 markup
- Keep JavaScript minimal and focused on interactivity
- Maintain the "glass-card" design aesthetic

### 3. Submitting Pull Requests

- **Fork the repository** and create a feature branch
- **Make your changes** following the code style
- **Test your changes** locally in multiple browsers
- **Submit a pull request** with a clear description of what you changed and why
- **Ensure all PRs reference an issue** (either create one or link to an existing one)

### 4. Code Review Process

- The core team reviews pull requests weekly
- We may request changes or clarifications
- Once approved, your contribution will be merged

## Community

This project follows the **Wikipedia Model** - radically transparent, collaborative, and for public benefit. We welcome contributors from all backgrounds, especially ethicists, habitat designers, geneticists, astrobiologists, and remote operations experts.

## Questions?

If you need help or have questions, open an issue or contact the project leads.
